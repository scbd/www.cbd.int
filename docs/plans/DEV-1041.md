# DEV-1041 — Session display of AI translations (block, marker, disclaimer)

**Status:** Plan (Jira IN PROGRESS). Branch `feature/DEV-1041-ai-translation-display` (from `master`).
**Type:** Additive, surgical UI change on the public meeting-session read view. Gated entirely on the
per-File `autoTranslated` flag: when it is absent/false, rendering and behaviour are byte-for-byte
identical to today.

## Intent

In the public meeting-session read view:

1. Split each intervention's `files[]` into two headed blocks — **Original statements** (Files with no
   `autoTranslated`) and **AI-translated statements** (`autoTranslated === true`).
2. Mark every translated row with a Font-Awesome v4 `fa-magic` + `fa-language` icon carrying a localized
   tooltip.
3. Gate translated-file downloads behind a localized disclaimer modal that contains the actual Download
   button. Originals keep the current direct-download anchor, unchanged.
4. Show a page-level Bootstrap `alert alert-warning` disclaimer banner whenever any *loaded* intervention
   carries an AI-translated File; hide it otherwise.

Non-translated statements must render exactly as today.

## Implementation steps

### Step 1 — Extract the per-File row into `file-row.vue` (no behaviour change)

Create `FileRow` with props `file: { type: Object, required: true }` and
`showPreviewAsButton: { type: Boolean, default: false }`. Move the desktop span, mobile span, and the
`!isPublic` eye-slash from `files-view.vue:7-31` into it verbatim. Move `getMimeConfig`, the `MIMES`
table, `langTextFilter`, and the `showPreview` import/method into `FileRow` (they are per-row concerns).
Keep the `<style scoped>` rules that target row internals (`.red/.blue/.green/.orange`, `.language`,
`d-md-none > .language`, `.btn-outline-dark:hover`) with the row.

`files-view.vue` then renders `<FileRow>` and keeps only the outer `.document-files` container + its
padding style.

**verify:** with an intervention that has only originals, the rendered DOM/behaviour is identical to
`master` (same icons, preview button, language label, eye-slash, download links, mobile layout). Diff
visually against current.

### Step 2 — Split `files-view.vue` into two blocks

Add two computeds deriving from the `files` prop. Because the prop is declared `type: Object`
(`files-view.vue:59`) but is used as an iterable list, normalize defensively:

```
originals()    { return this.fileList.filter(f => !f.autoTranslated) }
translations() { return this.fileList.filter(f =>  f.autoTranslated) }
// fileList(): Array.isArray(this.files) ? this.files : Object.values(this.files || {})
```

Render two headed blocks, each `v-for`-ing `FileRow` over its list:

- Originals block: heading `$t('Original statements')`, always rendered when `originals.length`.
- Translations block: heading `$t('AI-translated statements')`, rendered only when
  `translations.length`.

**Order within each block is the received order** — the current component does **not** sort by MIME.
The `priority` field in `MIMES` (`files-view.vue:44-53`) is defined but unused for display sorting.
"Preserving current MIME sort" therefore means *do not introduce a sort*: keep each block in the order
the API returns. Do not add ordering logic.

When only originals exist, suppress the "Original statements" heading is a design choice — **default:
keep the heading only when a translations block is also present**, so single-block interventions (the
overwhelming maintenance-era majority) look unchanged. (Confirm this reading against the ticket mockup;
flagged in Risks.)

**verify:** an intervention with 1 original + 1 translation shows two labelled blocks; an intervention
with only originals shows exactly today's layout (no stray heading, no empty translations block).

### Step 3 — Marker on translated rows (inside `FileRow`)

In `FileRow`, when `file.autoTranslated`, render the marker next to the icon: FA v4 `fa fa-magic`
followed by `fa fa-language` (no `fa-robot` exists in v4), wrapped in an element with
`:title="$t('AI-generated translation')"` for the localized tooltip. Gate with `v-if` so original rows
are untouched.

**verify:** translated rows show the magic+language marker with a tooltip on hover; original rows show
no marker; icons render (FA v4 present).

### Step 4 — Gated download in `FileRow`

Branch the click behaviour on `file.autoTranslated`:

- **Original (default):** keep the existing `<a target="_blank" :href="url">` direct-download anchor
  unchanged (`files-view.vue:11-16` desktop, `:23-25` mobile).
- **Translated:** replace the direct anchor with a click handler
  (`@click.prevent="showDisclaimer(file)"`) that opens the disclaimer dialog. The icon/language label
  stay, but navigation is intercepted — the actual download happens from the modal's button.

Import `showDisclaimer` from `./disclaimer-dialog.vue` into `FileRow`.

**verify:** clicking a translated file opens the modal and does NOT download directly; clicking an
original downloads immediately as before.

### Step 5 — New `disclaimer-dialog.vue`

Clone the `text-preview-dialog.vue` structure (`:116-134` `showPreview`, `:101-112` `mounted`,
`:63-72` `open`): a Bootstrap modal, mounted programmatically via `Vue.extend`, shown with
`$(el).modal('show')`, self-destroying on `hidden.bs.modal` (`$destroy` + `$el` remove). Export
`showDisclaimer(file)` that mounts the dialog with `propsData: { file, show: true }`.

Contents:

- Title: `$t('AI translation disclaimer')`.
- Body: the verbatim disclaimer text (see i18n key `aiDisclaimerText` below).
- Footer: a **Download** button — `<a class="btn btn-primary" :href="file.url" target="_blank"
  @click="close()">{{ $t('Download') }}</a>` — plus a Close button (`data-dismiss="modal"`).

Keep it minimal: no search/magnification machinery from the preview dialog.

**verify:** the modal shows the localized disclaimer and a Download button that fetches `file.url` in a
new tab; closing/downloading destroys the instance (no leaked DOM/`.modal-backdrop`).

### Step 6 — Page disclaimer banner in `view.vue`

Add a computed on `SessionsView`:

```
hasAiTranslations() {
  return (this.sessions || []).some(s =>
    (s.interventions || []).some(i =>
      hasTranslatedFile(i) ||
      (i.supersededChildren || []).some(hasTranslatedFile)))
}
// hasTranslatedFile(i): (i.files || []).some(f => f.autoTranslated) — normalize files as in Step 2
```

Render at the top of the template (above the sessions `v-for`, `view.vue:2-3`):

```
<div v-if="hasAiTranslations" class="alert alert-warning" role="alert">{{ $t('aiDisclaimerText') }}</div>
```

**Reactivity:** interventions load lazily per session; each session object is created with
`interventions: null` (`view.vue:107`), so the property is reactive and the assignment in
`loadInterventions` (`view.vue:152-154`) re-triggers the computed. Superseded children are included
because they carry their own `files` (`api.js:404-413`).

**verify:** the banner is absent on first render of a session with no translations; expanding/loading a
session that contains a translated file makes the banner appear; a meeting with zero translations never
shows it.

### Step 7 — i18n keys (English only; other UN langs fall back via `fallbackLocale:'en'`)

Add to `app/components/meetings/locales.js` under `messages.en`:

| Key | English value |
| --- | ------------- |
| `Original statements`        | `Original statements` |
| `AI-translated statements`   | `AI-translated statements` |
| `AI-generated translation`   | `AI-generated translation` |
| `AI translation disclaimer`  | `AI translation disclaimer` |
| `Download`                   | `Download` |
| `aiDisclaimerText`           | *(verbatim)* `AI-generated translations of statements are made available to enhance accessibility. The Secretariat does not guarantee the accuracy, completeness, or reliability of these translations. Only the respective original submitted statements shall be considered authoritative.` |

Short labels use natural-language keys to match the existing convention (`locales.js:7-10`); the long
paragraph uses the semantic key `aiDisclaimerText` to avoid an unwieldy key (minor, intentional
deviation).

**vue-i18n scoping note (verify during Steps 3/5/6):** the global instance from `app.js:105-112` is
created with **no `messages`** — the meetings strings live only in the component-level `i18n` option
that imports `locales.js` (as `intervention-row.vue:73,96` does). In vue-i18n v8 (`boot.js:56`), a
component that declares its own `i18n` option gets a *local* root and does **not** inherit a parent's
component-level messages. Therefore each component that renders these new keys must attach the meetings
locale itself:

- `file-row.vue`, `disclaimer-dialog.vue`, and `view.vue` must each `import i18n from '../locales.js'`
  and add `i18n,` to their component options (same one-liner pattern as `intervention-row.vue`).
- `files-view.vue` needs it too for the block headings.

If manual testing shows a raw key rendering instead of English text, the missing `i18n` option on that
component is the cause — add it. This is the concrete DEV-1036 dependency.

**verify:** headings, tooltip, modal, and banner all render English text (not raw keys) with
`locale='en'`; switching `locale` to a language with no messages still shows English (fallback).

## Risks / assumptions

1. **DEV-1010 draft contract (scbd/gaia#1083).** The `autoTranslated` File contract is unmerged and
   could shift (field name, object placement, whether `translations[]` is stripped publicly). Step 0 is
   a hard gate; if the live/fixture response disagrees, stop and reconcile before coding. All UI is
   gated on `autoTranslated`, so a missing flag degrades safely to today's behaviour.
2. **vue-i18n readiness (DEV-1036 / commit c295ad70).** Global `$t` and `fallbackLocale:'en'` are wired,
   but the global instance carries no messages; new-key rendering depends on each component attaching the
   `locales.js` `i18n` option (Step 7 note). Verified pattern exists in `intervention-row.vue`.
3. **`files` prop typed `Object` (`files-view.vue:59`).** Treated as a list today. Normalize via
   `Array.isArray ? files : Object.values(files||{})` before filtering so the split is robust to either
   shape. Verify the real payload shape in Step 0.
4. **No active MIME sort.** `MIMES.priority` is unused for display ordering; blocks preserve received
   order. Do not add sorting.
5. **Single-block heading behaviour.** Plan keeps the "Original statements" heading only when a
   translations block is also present, so today's single-block view is visually unchanged. If the ticket
   mockup wants the heading always shown, adjust Step 2 — flag for the human before implementing.
6. **No frontend test framework** (per ticket); verification is manual/browser-based as specified in each
   `verify:` above.

## Boundaries honoured

Additive and surgical: two new components + block split + banner + i18n keys. No refactor of untouched
code, no new abstraction beyond the single shared `FileRow` the ticket mandates, no backend changes
(none possible here). Downloads for originals are left exactly as-is.
